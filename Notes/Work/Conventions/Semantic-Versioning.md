# Semantic Versioning 
## Источник
- https://semver.org/
## Описание
Семантическое версионирование (semver) — это система нумерации версий программного обеспечения, которая помогает разработчикам и пользователям понимать изменения в программном обеспечении на основе его версии. Семантическое версионирование использует формат MAJOR.MINOR.PATCH, где:
- MAJOR (главная версия): увеличивается, когда вносятся несовместимые изменения
- MINOR (второстепенная версия): увеличивается, когда добавляются новые функции, которые обратно совместимы
- PATCH (патч): увеличивается, когда вносятся исправления ошибок, которые обратно совместимы

## Регулярные выражения
- Полное соответствие версии: `^(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)(?:-((?:0|[1-9]\d*|[0-9A-Za-z-][0-9A-Za-z-]*)(?:\.(?:0|[1-9]\d*|[0-9A-Za-z-][0-9A-Za-z-]*))*))?(?:\+([0-9A-Za-z-]+(?:\.[0-9A-Za-z-]+)*))?$`
- Основные версии: `^(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)`
- Префикс версии: `^(v|version-)?(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)`
